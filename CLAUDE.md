## Project Overview

Dashboard for querying ITBI (real estate transfer tax) transactions in São Paulo, 2006–2026. Data is scraped monthly from the official Prefeitura SP source, stored in SQLite, and served via a FastAPI backend with a single-file SPA frontend.

Production: **https://itbisp.mooo.com** (Oracle Cloud VM · `ubuntu@137.131.160.254`)

## Commands

### Local development

```powershell
# Install dependencies
cd D:\itbi-dashboard\backend
pip install -r requirements.txt

# Create local dev flag (disables heavy background tasks)
New-Item backend\.dev_local -ItemType File

# Populate database (quick test — recent years only)
python scraper.py --anos 2024 2025 2026

# Populate database (first time — downloads all years, 10-20 min)
python scraper.py

# Force re-download even if file hash unchanged
python scraper.py --forcar

# Start API (dev mode with reload — no login required on localhost)
python -m uvicorn main:app --reload
# → http://localhost:8000  (API)
# → http://localhost:8000/docs  (Swagger UI)
```

## Architecture

```
backend/
  main.py        — FastAPI app; serves API + mounts frontend as static files
  scraper.py     — Downloads XLSX files from Prefeitura SP and populates SQLite
  scraper_csv.py — CSV variant of the scraper
  exportar.py    — Excel/PDF export helpers (called by main.py export routes)
  geo.py         — Geocoding (map feature, currently disabled in API)
  auth.py        — JWT authentication
  billing.py     — Stripe billing integration
  itbi.db        — SQLite database (not committed; generated by scraper)
  .dev_local     — Flag file: disables iptu background task locally (not committed)
  cache/         — Downloaded XLSX files cached locally (not committed)
frontend/
  index.html     — Single-file SPA (vanilla JS + Chart.js)
backup-v1/       — Snapshot of VM files before v2 deploy (rollback source)
deploy-v2.ps1    — Deploys v2 files to VM (uses $VM variable, not $HOST)
rollback-v1.ps1  — Restores v1 from backup-v1/ to VM
```

### Data flow

1. `scraper.py` downloads yearly XLSX files from Prefeitura SP URLs (see `ARQUIVOS` dict)
2. Each XLSX has monthly sheets; `processar_xlsx()` reads all sheets via openpyxl
3. Column names vary wildly across years — `COL_MAP` dict normalizes ~80 variants to canonical names
4. Data is saved to `transacoes` table (delete-then-insert per year); processed file hashes stored in `arquivos_processados` to skip unchanged files
5. `main.py` startup creates indexes, enables WAL mode, and pre-warms the `/api/resumo` cache in a background thread
6. A secondary `iptu` table is populated lazily from `transacoes`, keyed by `sql_terreno` (IPTU property code)

### Key implementation details

- **SQL injection protection**: `buscar_transacoes` uses a whitelist (`COLS_PERMITIDAS`) for sort column; all filters use parameterized queries
- **Multi-value filters**: `_multi_like()` splits comma-separated values into OR clauses (e.g., `bairro=pinheiros,vila madalena`)
- **CEP/SQL normalization**: Strips punctuation before comparison so `02806-000` matches `02806000`
- **`/api/resumo` cache**: Simple dict with 5-minute TTL; cache key is the WHERE clause + params string
- **Autocomplete cache**: All distinct values loaded into Python lists at startup (`_ac_logradouros`, `_ac_bairros`, etc.) — filtering in Python with no DB hits per keystroke; fallback to DB if cache not yet loaded
- **Autocomplete pagination**: `_ac_page()` helper; endpoints accept `offset` param; response includes `{items, has_more, total}`
- **Prewarm**: `_prewarm()` calls `resumo()` directly (not via HTTP) to ensure full result including `por_mes` and `faixas_valor` is cached for default (3yr), all years, and old years ranges
- **Frontend served by FastAPI**: `StaticFiles` mount at `/` — must be registered last so it doesn't shadow API routes
- **No-cache header**: Middleware adds `Cache-Control: no-cache` to HTML responses to prevent stale browser cache
- **Auth bypass for localhost**: Middleware checks `request.client.host in ("127.0.0.1", "::1")` — no token needed locally
- **`.dev_local` flag**: Disables the `_popular_iptu_lazy` background thread locally to avoid slow startup
- **Frontend auth bypass**: `IS_LOCAL` check in JS; login.html redirects to index.html if on localhost
- **Tag-based filter inputs**: `_initTagInput()` — multi-select chips with autocomplete, infinite scroll (8 items + load more on scroll), preview-on-focus for logradouro/bairro/cep/sql
- **KPI cards update on search**: When `filtersChanged=true`, `/api/resumo` is fetched with current filters and `renderKPI()` updates the top cards
- **KPI cards update on selection**: `applySelectionToKPI()` called on row check/uncheck — shows selected rows stats; reverts to `lastResumo.geral` when deselected
- **Glass popup (selpop)**: Floating summary popup shows when rows are selected; separate from export bar
